# Changelog

## 2.11.0 — 2026-04-24

주요 주제: **문서 간 이동 개선** — 미리보기 탭 뒤로/앞으로 네비게이션 추가, 우측 링크 패널이 실제로 문서를 찾도록 수정.

### 추가

- **뒤로/앞으로 네비게이션** — 툴바 로고 옆에 ← / → 버튼. `Alt+←` / `Alt+→` 단축키. 사이드바·링크 패널·위키링크로 미리보기 탭이 갱신될 때마다 히스토리에 기록되고, 뒤로 버튼으로 이전에 보던 문서로 복귀. 히스토리 끝에 도달하면 버튼이 자동 비활성화됨.
- **우측 링크 패널에서 경로 포함 위키링크 해석** — `[[subfolder/file]]` 같은 경로 지정 위키링크가 이제 제대로 해석됨. 기존에는 `findFile`이 파일명만 비교해서 같은 이름 파일이 여러 폴더에 있으면 구분 못 하거나 아예 못 찾았음.

### 수정

- **우측 링크 패널 "참조하는 문서 없음"** — 배지에는 링크 개수가 찍히는데 리스트는 비어 보이던 문제. 원인은 두 가지: (1) 파일이 등록된 프로젝트 폴더 밖일 때 `projectPath`가 빈 문자열이라 해석을 아예 스킵, (2) Windows 경로 대소문자/구분자 차이(`C:\` vs `c:/`)로 프로젝트 매칭이 실패.
  - `RightPanel` 이 `projectPath` 없으면 현재 파일 디렉터리를 탐색 루트로 폴백, 그것도 없으면 이름만이라도 노출.
  - `MarkdownEditor` / `LiveEditor` 의 프로젝트 매칭을 정규화 비교(`replace(/\\\\/g, '/').toLowerCase()` + `file === root || file.startsWith(root + '/')`)로 교체. 경계 체크(`+ '/'`)로 `C:\\foo` 가 `C:\\foobar` 에 오매칭되는 케이스도 방지.

### 내부

- 저장소에 `navHistory`, `navIndex`, `_skipNavPush` 상태 추가. `openTab` 이 `isPreview=true` 로 새 탭을 만들거나 교체할 때마다 히스토리 푸시 — 뒤로/앞으로로 트리거된 오픈은 플래그로 스킵.
- 커스텀 이벤트 `nav:go` 로 툴바 버튼 → App 루트 리스너 → `openFile` 경로 연결.

주요 주제: **에디터 문법 색상 하이라이트 (VSCode 스타일)** — 편집 창에서 마크다운 토큰마다 색이 다르게 나오도록 풀 팔레트 적용.

### 추가

- **VSCode/GitHub Primer 스타일 마크다운 하이라이트** — `LiveEditor` 에 라이트/다크 각각의 `HighlightStyle` 을 정의해 `syntaxHighlighting` 으로 연결. 기존에는 라이트 모드에서 사실상 색이 없고 다크 모드는 `oneDark` 기본 팔레트만 적용돼 제목/본문 구분이 약했음.
- **토큰별 색 적용** — 제목(H1~H6), `**굵게**`, `*기울임*`, `~~취소선~~`, 링크(`[text](url)`), 인라인/블록 코드, `> 인용`, `---` 구분선, `<details>` 같은 HTML 태그에 개별 색 지정.
- **마크업 문자 뮤티드화** — `#`, `**`, `*`, `` ` ``, `>`, `-`, `[`, `]` 등의 구문용 문자(`t.processingInstruction`, `t.meta`)를 본문보다 흐린 회색으로 처리해 "기호는 뒤로, 내용은 앞으로" 읽히게 함.
- **다크 모드 중첩 처리** — `oneDark` 는 배경/캐럿/선택 영역(chrome)만 담당하고, 그 위에 마크다운 전용 색 레이어(`mdHighlightDark`)를 덮어써 마크다운 토큰 색만 바뀌고 에디터 외관은 기존과 동일.

### 참고

- 폰트 크기는 제목이어도 변경하지 않음 — VSCode 에디터 동작과 일치, 그리고 분할 뷰 스크롤 싱크 (`data-line` 기반 offset 매핑)가 변동 폭 라인 높이에 취약하기 때문.
- 기존 `noUnderlineLinks` 스타일의 "YAML frontmatter 배열이 링크로 오인되어 밑줄 쳐지던 현상" 우회는 새 스타일의 `{ tag: t.link, textDecoration: 'none' }` 룰로 자연스럽게 흡수됨.

## 2.9.0 — 2026-04-23

주요 주제: **네이티브 다이얼로그 제거** — Alt+Tab을 눌러야 입력이 복구되던 고질적 먹통 버그의 근본 원인을 제거.

### 수정 — 입력 먹통 (freeze) 버그 근본 해결

- **원인 확정** — Windows + Electron + 커스텀 `titleBarOverlay` 조합에서 `window.confirm` / `window.prompt` / `window.alert`(네이티브 다이얼로그)가 닫힌 뒤 BrowserWindow가 OS 레벨에서 "비활성" 포커스 상태로 고착. 렌더러의 키 입력이 먹혀 조합 중인 한글이 사라지거나 타이핑이 무시됨. Alt+Tab / 창 리사이즈로만 복구.
- **해결** — 앱 전체에서 네이티브 다이얼로그 호출을 제거하고 인앱 모달로 교체. 새로운 imperative API (`alert()` / `confirm()` / `prompt()`)는 Promise를 반환하고 BrowserWindow focus 상태를 건드리지 않음 → 포커스 고착이 원천 차단.
- **대상** — `window.confirm` 15곳, `window.prompt` 1곳, `alert` 27곳 (43 콜사이트).

### 추가

- **`src/utils/dialog.ts`** — `alert/confirm/prompt` imperative API. 문자열 단축 형 또는 옵션 객체로 호출 가능 (`variant: 'danger'`, 커스텀 라벨, 제목 등).
- **`src/components/DialogHost.tsx`** — App 루트에 상주하는 모달 호스트. ESC로 취소, Enter로 확인, 배경 클릭으로 닫힘, 다크모드 대응, 큐 기반으로 다이얼로그 중첩 안전 처리.

## 2.7.4 — 2026-04-22

### 수정 — Word → MD 표 변환 시 셀 내부 개행 때문에 표가 깨지던 문제

- **원인** — mammoth는 Word 셀 내용을 `<p>`로 감싼 HTML로 내놓고, turndown-plugin-gfm의 `cell()` 함수는 내용의 개행을 정리하지 않음. 결과적으로 셀마다 문단이 여러 줄로 쪼개져 출력 → GFM 파이프 문법이 깨지고 프리뷰에서 표로 인식되지 않음.
- **해결** — turndown 전에 HTML 후처리를 한 단계 더 추가. `<td>`/`<th>` 내부의 `<p>`를 내용 + `<br>`로 평탄화하고, 이어지는 공백·개행을 한 칸으로 압축. 셀 안의 `|` 문자는 `\|`로 이스케이프. 단일 라인 파이프 표가 되어 렌더링이 정상화됨.

## 2.7.3 — 2026-04-22

### 수정 — Word → MD 가져오기 테이블이 여전히 사라지던 문제 (v2.7.2 불완전)

- **원인** — turndown-plugin-gfm의 파이프 표 변환 규칙은 **첫 행이 전부 `<th>`**일 때만 매칭. 그렇지 않은 표는 `turndown.keep()`으로 raw `<table>` HTML로 남음. Word 문서 대부분은 "Table Header" 스타일을 명시하지 않기 때문에 mammoth가 모든 셀을 `<td>`로 내놓음 → 파이프 표로 변환되지 않고, raw HTML은 뷰어의 sanitize에 걸려 보이지 않게 됨.
- **해결** — turndown 실행 전에 HTML을 후처리해서, `<th>`가 없는 각 표의 첫 `<tr>`에 있는 `<td>`를 모두 `<th>`로 승격. 이로써 첫 행이 헤더인 것처럼 취급되어 GFM 파이프 표(`|---|` 구분선 포함)로 깔끔하게 변환됨.

## 2.7.2 — 2026-04-22

### 수정 — Word → MD 가져오기에서 테이블이 사라지던 문제

- **원인** — `mammoth.convertToMarkdown`은 공식 문서에서 "tables and some other features are not supported"로 명시된 제한된 변환기. 표는 그냥 드롭되어 빈 공간으로 남았음.
- **해결** — 경로를 `mammoth.convertToHtml` → `turndown` + `turndown-plugin-gfm`으로 교체. mammoth가 내놓은 `<table>` 마크업을 GFM 파이프 표(헤더 구분선 포함)로 변환. 헤딩/목록/인라인 서식도 동일하게 유지.
- **부작용** — 뉴 deps: `turndown` + `turndown-plugin-gfm` + `@types/turndown`.

## 2.7.0 — 2026-04-22

주요 주제: **VSCode급 실시간 프로젝트 뷰** — 외부 편집(Claude Code · 터미널 · 다른 에디터)이 즉시 파일 트리와 에디터에 반영됩니다.

### 추가

- **외부 Git 작업 자동 반영** — `.git/index`, `.git/HEAD`, `.git/refs/heads/` 를 전용 워처로 감시. 터미널에서 `git add` / `git commit` / 브랜치 전환해도 배지가 실시간으로 갱신. Windows에서 chokidar가 원자적 교체를 놓치는 케이스 대비로 3초 폴링 안전망도 함께 동작.
- **mtime 기반 "최근 변경" 점** — Git으로 관리하지 않는 폴더에서도 60초 이내에 수정된 파일은 트리에 회색 점으로 표시. 외부 편집이 일어났다는 신호가 Git 유무와 무관하게 잡힘.
- **변경 순간 플래시 애니메이션** — 외부에서 파일이 생성·수정·삭제되면 해당 트리 행이 노란색으로 3초간 점멸. 어느 파일이 방금 움직였는지 한눈에.

### 동작 변경

- **미저장 변경이 없을 때 자동 리로드** — 열린 탭이 dirty가 아닌 상태에서 파일이 외부에서 바뀌면 "외부에서 변경됨" 배너 없이 조용히 리로드. dirty일 때는 기존처럼 배너를 띄워 사용자가 선택.
- **디렉토리 워처가 `change` 이벤트도 감지** — 이전에는 add/unlink만 감시해 기존 파일의 내용 변경은 트리 리로드가 트리거되지 않았음.

### 내부

- `fs:watchGit` / `fs:unwatchGit` IPC 추가. `onDirChanged` 이벤트가 변경된 경로 목록을 함께 전달하도록 확장 (후방 호환: 기존 1-인자 콜백도 그대로 동작).
- `FileNode.mtime` 필드 추가. `readDirRecursive`가 각 파일의 `mtimeMs`를 포함해 반환.

## 2.6.0 — 2026-04-22

- **MD → PDF 내보내기** — 마크다운을 자체 스타일로 렌더해 Electron `printToPDF`로 생성 (A4, 여백 0.5in). 표/코드/인용/이미지 모두 반영.

## 2.3.0 — 2026-04-21

주요 주제: **Git UX 개선**, **텍스트 파일 지원 확대**, **미리보기 탭 버그 수정**

### 추가 — Git

- **Pull 후 변경 요약 패널** — 받기(Pull)로 내려온 커밋 목록과 변경된 파일이 Git 탭 상단에 인라인으로 표시. 파일 이름 클릭 시 에디터에서 바로 열림. 변경 없으면 "이미 최신 상태" 토스트만 표시.
- **Push 타임아웃 확대** — 30초 → 5분. 첫 인증(GCM 브라우저 팝업) 및 큰 pack 전송 시 조기 타임아웃 방지.
- **Push 실패 후 실제 성공 자동 감지** — 타임아웃/네트워크 오류로 실패처럼 보여도 `git fetch + rev-list`로 원격 기준 ahead=0이면 성공으로 전환. "업로드가 이미 완료된 것으로 확인되었습니다" 메시지 표시.
- **Push 후 항상 새로고침** — 성공/실패 관계없이 ahead 배지를 즉시 재조회. "실패했다고 떴는데 배지는 그대로"여서 사용자가 중복 Push → merge 커밋이 생기는 악순환 차단.
- **`docs/git-guide.md`** — 비개발자용 Git 사용 가이드 신규 (용어집, 평상시 흐름, 되돌리기 3종 비교, 병합 커밋의 정체, 영어 오류 번역표, 버튼↔git 명령 매칭표).

### 추가 — 텍스트 파일 지원

- `.json`, `.yml`, `.yaml`, `.xml`, `.csv`, JS/TS 계열, Python/Go/Rust 등 주요 언어, 쉘·설정 파일(`.conf`, `.cfg`, `.properties` 등) 확장자를 텍스트로 열 수 있도록 `TEXT_EXTS` 확장. (기존엔 `.txt/.log/.ini/.env/.toml`만 지원)

### 수정

- **미리보기 탭 재사용 시 스크롤이 맨 위로 리셋되지 않던 버그** — 이전 파일에서 디바운스된 스크롤 저장 타이머가 새 파일의 `scrollPos`를 덮는 레이스 차단. 파일 전환 시 대기 중인 타이머를 취소하고, 같은 `tab.id`에서 `filePath`만 바뀐 경우(미리보기 재사용) 강제로 `scrollTop=0`.
