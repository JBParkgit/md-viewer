# Changelog

## 2.20.0 — 2026-04-26

주요 주제: **상단 검색창을 Word 스타일 인라인 명령 팔레트로 통합**.

### 추가

- **툴바 검색창이 실제 입력 가능 (`Toolbar.tsx`, `CommandPalette.tsx`)** — 기존에 클릭하면 모달이 뜨던 "Search" 버튼을 진짜 `<input>` 으로 교체. 포커스되거나 글자를 입력하는 즉시 입력창 바로 아래에 명령 팔레트가 인라인으로 펼쳐지고, 빈 상태에서는 "추천 동작" 헤더와 함께 자주 쓰는 액션들이 노출됨. Word 의 "작업, 텍스트, 도움말 등을 검색해 보세요." 패러다임과 동일.
  - `↑`/`↓` 결과 이동, `Enter` 실행, `Esc` 닫기 모두 입력창에 포커스가 있는 상태에서 동작.
  - `Ctrl+K` 는 입력창에 포커스 + 전체 선택으로 매핑 (모달 별도 띄우지 않음).
  - 입력창 바깥 클릭 시 자동으로 닫힘 (입력창과 팔레트 패널은 클릭 영역에서 제외).
  - 팔레트가 자체적으로 닫힐 때(아이템 실행 / Esc / 외부 클릭) 입력창의 값을 자동으로 초기화.
- **글꼴 선택을 보기 메뉴 서브메뉴로 (`Toolbar.tsx`)** — 툴바의 글꼴 셀렉트 박스를 제거하고, 보기 메뉴 안에 "글꼴 ▶" 항목을 추가. 호버하면 옆으로 7가지 글꼴(시스템 기본 / Pretendard / Noto Sans KR / 나눔고딕 / 나눔명조 / 맑은 고딕 / 굴림)이 펼쳐지는 1단계 서브메뉴로 정리. 현재 글꼴은 ✓ 로 표시.

### 내부

- `MenuItem` 타입에 `submenu?: MenuItem[]` 필드 추가. 상위 메뉴를 다른 곳으로 옮기면 서브메뉴가 자동으로 닫히도록 `openSubIdx` state 와 `openIdx` 변경 effect 로 관리.
- `CommandPalette` 에 `mode: 'inline' | 'modal'` 필드 추가. inline 모드는 백드롭/자체 입력창 없이 anchor 아래에 패널만 렌더, query 는 외부(`command-palette:set-query` 이벤트)에서 주입. modal 모드는 호환성용으로 유지.
- 새 이벤트: `command-palette:set-query`, `command-palette:close`, `command-palette:closed`.

## 2.18.0 — 2026-04-25

주요 주제: **최근 받은 변경 파일을 한눈에 보기 (트리 배지 + 다시 보기 버튼)**.

### 추가

- **파일 트리에 "📥 받기 배지" (`FileTree.tsx`)** — 가장 최근 Pull 로 변경된 파일에 보라색 ↓ 아이콘이 자동으로 붙음. 기존 git 상태 배지(M/A/D, 주황/초록/빨강)와 색·모양으로 명확히 구분돼 "내가 편집한 것" vs "받은 것" 을 한눈에 식별 가능. 다음 Pull 이 오면 새 목록으로 자동 교체.
- **닫힌 폴더 집계 배지** — 폴더가 접혀 있어도 그 폴더 안에 받은 파일이 몇 개 있는지 보라색 카운트로 노출 (`📥 3`). 깊은 디렉터리 구조에서도 어디에 변경이 있는지 즉시 보임.
- **프로젝트 헤더에 "📥 N 다시 보기" 버튼 (`ProjectTree.tsx`)** — Pull 결과 다이얼로그를 닫은 뒤에도 같은 변경 파일 목록을 다시 열 수 있도록 헤더에 작은 보라색 카운트 버튼 추가. 클릭 시 저장된 스냅샷으로 다이얼로그가 즉시 다시 열림 (커밋 목록은 비어있게 표시되지만 파일 목록과 좌우 비교는 그대로 동작).

### 내부

- **`isFileInPullRange` / `getDirPullCount` 헬퍼 추가 (`FileTree.tsx`)** — 기존 `getGitDot` / `getDirGitStatus` 와 같은 모양의 두 함수. `lastPullByProject` 의 파일 목록을 파일/폴더 단위로 매칭. 경로 정규화는 git status 매칭과 동일한 방식.

## 2.17.2 — 2026-04-25

### 변경

- **인쇄는 항상 라이트 테마로 (`PrintPreviewModal.tsx`)** — 사용자가 다크 모드를 쓰고 있어도 PDF 생성·실제 인쇄 둘 다 라이트 테마로 출력. CSS `@media print` 만으로는 Tailwind `dark:` 유틸의 specificity 를 모두 덮기 어려워, IPC 호출 직전에 `html` 의 `dark` 클래스를 잠시 떼고 (두 프레임 대기로 라이트 페인트 보장) 호출이 끝나면 즉시 복원하는 방식. 모달 자체는 잠깐 라이트로 깜빡이지만 < 100ms 라 거의 인지 불가. 모달을 도중에 닫아도 `dark` 클래스가 복원되도록 cleanup 처리.

## 2.17.1 — 2026-04-25

### 수정

- **인쇄 미리보기에서 본문이 오른쪽으로 치우치고, 페이지 일부가 잘리고, 다크 테마 글씨가 안 보이던 문제 (`index.css`)** — v2.17.0 의 `@media print` 규칙이 `position: absolute; width: 100%` 로 인쇄 영역을 화면 viewport 기준으로 잡아 안쪽 `mx-auto` 가 화면 가운데로 정렬됐고, A4 종이는 그 결과의 일부만 잘라 보여 한쪽으로 치우쳐 보였음. 또한 absolute 가 자연 흐름을 막아 두 번째 페이지 이후가 잘렸고, `html.dark` 에서 회색 텍스트가 그대로 인쇄돼 종이에 거의 안 보였음.
  - `:has()` 셀렉터로 print 타겟의 조상만 골라 flex/width/height 제약을 모두 풀고, 형제(toolbar/sidebar/tab bar/...)는 `display: none`. 타겟은 자연 흐름에 두어 Chromium 인쇄 엔진이 알아서 페이지 분할.
  - 다크 테마용 라이트 강제: `[data-print-target] *` 에 `color: black`, 비-코드 컨테이너에 `background: transparent`. 코드 블록/인용 등 의도된 배경은 보존.
  - `print-color-adjust: exact` 로 syntax highlighting 배경도 종이에 그대로 출력되게 함.

## 2.17.0 — 2026-04-25

주요 주제: **인쇄 미리보기 모달 (PDF 기반)**.

### 추가

- **인쇄 미리보기 모달 (`PrintPreviewModal.tsx`)** — 🖨️ 인쇄 버튼 또는 `Ctrl+P` 클릭 시 더 이상 OS 인쇄 다이얼로그가 즉시 뜨지 않고, 앱 내부에서 **실제 인쇄될 PDF를 미리 보여주는 모달**이 먼저 열림. 사용자가 페이지 분할·레이아웃을 확인한 뒤 "🖨️ 인쇄" 버튼을 눌러야 OS 인쇄 다이얼로그로 넘어감. 줌 50~300% 지원, 키보드 Esc 로 닫기.
- **`webContents.printToPDF` IPC (`electron/main.ts`)** — `printPreview:generate` 핸들러가 A4·배경 포함·기본 마진 0.5인치로 PDF 생성 후 base64 로 렌더러에 전달. `printPreview:print` 는 `webContents.print({ silent: false })` 로 OS 인쇄 다이얼로그 호출. 두 쪽 모두 동일한 `@media print` CSS 를 거치므로 미리보기 ↔ 종이 결과가 1:1 일치.

### 변경

- **`@media print` CSS 페이지네이션 친화 개선 (`index.css`)** — 기존 `position: absolute; inset: 0` 가 인쇄 영역을 뷰포트 크기에 가둬 PDF 첫 페이지 이후가 잘리던 문제. `width: 100%; height: auto; left: 0; top: 0` 로 바꿔 콘텐츠 자연스럽게 흐르도록 수정. Chromium 의 인쇄 엔진이 알아서 페이지 분할 처리.

### 수정 — 입력 먹통 (freeze) 버그 근본 해결

- **원인 확정** — Windows + Electron + 커스텀 `titleBarOverlay` 조합에서 `window.confirm` / `window.prompt` / `window.alert`(네이티브 다이얼로그)가 닫힌 뒤 BrowserWindow가 OS 레벨에서 "비활성" 포커스 상태로 고착. 렌더러의 키 입력이 먹혀 조합 중인 한글이 사라지거나 타이핑이 무시됨. Alt+Tab / 창 리사이즈로만 복구.

## 2.7.4 — 2026-04-22

### 수정 — Word → MD 표 변환 시 셀 내부 개행 때문에 표가 깨지던 문제

- **원인** — mammoth는 Word 셀 내용을 `<p>`로 감싼 HTML로 내놓고, turndown-plugin-gfm의 `cell()` 함수는 내용의 개행을 정리하지 않음. 결과적으로 셀마다 문단이 여러 줄로 쪼개져 출력 → GFM 파이프 문법이 깨지고 프리뷰에서 표로 인식되지 않음.

## 2.7.3 — 2026-04-22

### 수정 — Word → MD 가져오기 테이블이 여전히 사라지던 문제 (v2.7.2 불완전)

- **원인** — turndown-plugin-gfm의 파이프 표 변환 규칙은 **첫 행이 전부 `<th>`**일 때만 매칭. 그렇지 않은 표는 `turndown.keep()`으로 raw `<table>` HTML로 남음. Word 문서 대부분은 "Table Header" 스타일을 명시하지 않기 때문에 mammoth가 모든 셀을 `<td>`로 내놓음 → 파이프 표로 변환되지 않고, raw HTML은 뷰어의 sanitize에 걸려 보이지 않게 됨.
- **해결** — turndown 실행 전에 HTML을 후처리해서, `<th>`가 없는 각 표의 첫 `<tr>`에 있는 `<td>`를 모두 `<th>`로 승격. 이로써 첫 행이 헤더인 것처럼 취급되어 GFM 파이프 표(`|---|` 구분선 포함)로 깔끔하게 변환됨.

### 추가 — Git & 파일 이력

- 파일 우클릭 → **`📜 이 파일의 이력`** — 좌측 커밋 목록 / 우측 시점별 내용 미리보기 / `↩️ 이 시점으로 되돌리기`
- 새 IPC: `git:fileLog`, `git:fileShow`, `git:checkoutFileAtCommit`
- `gitExec`에 **UTF-8 강제 옵션** (`-c i18n.logOutputEncoding=UTF-8 -c i18n.commitEncoding=UTF-8` + `LC_ALL=C.UTF-8`) → 한글 작성자명·커밋 메시지·파일 내용 mojibake 해결

### 추가 — 저장 & UX 피드백

- 저장 완료 시 **`저장됨 ✓`** 초록 토스트, 변경 없을 때 Ctrl+S는 **`이미 저장됨 ✓`** 회색 토스트로 구분
- `file-saved` 글로벌 이벤트 → ProjectTree가 자동으로 git status 갱신 (M/A/D 표시 즉시 반영, 수동 새로고침 불필요)
- 에디터 하단 **상태 바** — 단어 수 / 문자 수 / 공백 제외 수 + 파일명. CJK(한·중·일) 문자는 1단어 단위로 집계
- 폴더 펼침 상태(`openDirs`)를 electron-store에 영속화 — 사이드바 탭 전환이나 앱 재시작 후에도 유지
- 워크플로우 시작이나 태그 추가 시 "파일이 외부에서 변경됨" 배너가 뜨는 문제 해결 (메인 프로세스에서 self-write 추적으로 억제)

### 변경
