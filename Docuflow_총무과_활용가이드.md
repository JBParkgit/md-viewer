# Docuflow 활용 가이드

## 병원 총무과를 위한 문서 관리 & 업무 협업 가이드

---

## 1. Docuflow란?

Docuflow는 마크다운(.md) 문서를 중심으로 파일을 관리하고, 편집하고, 팀과 공유할 수 있는 데스크톱 프로그램입니다.

총무과의 각종 공문서, 회의록, 업무 지시, 보고서를 한 곳에서 작성·보관·검색할 수 있습니다.

### 왜 Docuflow를 쓰나요?

| 기존 방식 | Docuflow 방식 |
|----------|-------------|
| Word 파일 → 이메일 전달 → 수정 → 재전달 | 공유 폴더 + Git으로 실시간 동기화 |
| 파일이 여러 버전으로 흩어짐 | 변경 이력이 자동 기록됨 |
| 어떤 파일이 최신인지 불분명 | 커밋 메시지로 변경 내용 추적 가능 |
| 검색이 어려움 | 파일명·내용 전문 검색 지원 |

### 주요 기능

- 마크다운 편집기 + 실시간 미리보기
- 프로젝트(폴더) 단위 파일 관리
- 칸반 보드로 업무 진행 현황 관리
- 태그 기반 문서 분류 및 검색
- Git 연동 (저장/올리기/받기)
- Word(.docx), PDF 미리보기
- 전문 검색 (마크다운 + 워드 문서 내용)

### 사이드바 탭 구성

| 탭 | 기능 |
|----|------|
| 프로젝트 | 폴더 구조 탐색, 파일 생성/이동 |
| 즐겨찾기 | 자주 쓰는 파일 빠른 접근 |
| 최근 파일 | 최근에 열어본 파일 목록 |
| 태그 | 문서 분류 및 필터링 |
| 갤러리 | 사진·스캔 파일 모아보기 |
| 칸반 보드 | 업무 카드 관리 |
| Git | 버전 관리 및 팀 동기화 |

## 2. 설치 및 첫 실행

### 설치

1. `Docuflow` 폴더를 PC의 원하는 위치에 복사
2. `Docuflow.exe` 실행
3. 바탕화면 바로가기 생성 권장

### 첫 실행 설정

1. 좌측 상단 **"프로젝트 추가"** 버튼 클릭
2. 총무과 공유 폴더(또는 개인 폴더) 선택
3. 좌측 트리에서 파일 탐색 시작

## 3. 총무과 폴더 구조

### 추천 폴더 구조

```
총무과_업무/
  ├── 공문서/
  │   ├── 수신/
  │   │   └── 2026/
  │   └── 발신/
  │       └── 2026/
  ├── 회의록/
  │   └── 2026/
  ├── 보고서/
  │   ├── 월간보고/
  │   ├── 주간보고/
  │   └── 현안보고/
  ├── 계약_구매/
  │   ├── 용역계약/
  │   ├── 물품구매/
  │   └── 시설유지보수/
  ├── 인사_노무/
  │   ├── 채용/
  │   └── 교육연수/
  ├── 시설_안전/
  │   ├── 점검일지/
  │   └── 민원/
  ├── 예산_회계/
  │   └── 2026/
  └── 서식_템플릿/
```

### 담당자별 역할

| 담당자 | 주요 폴더 | 주요 업무 |
|--------|----------|----------|
| **과장** | 보고서/, 공문서/ | 결재·승인, 현안 보고, 주요 공문 |
| **팀원 A** | 공문서/, 계약_구매/ | 공문 접수·발송, 구매·계약 관련 문서 |
| **팀원 B** | 회의록/, 인사_노무/ | 회의록 작성, 채용·교육 관련 문서 |
| **팀원 C** | 시설_안전/, 예산_회계/ | 시설 점검일지, 예산 집행 내역 |

> **충돌 방지 팁:** 같은 파일을 동시에 수정하면 Git 충돌이 발생할 수 있습니다. 담당 폴더를 나누어 작업하고, 수정 전 반드시 **Pull(받기)**을 먼저 하세요.

## 4. 마크다운 기본 사용법

### 자주 쓰는 문법

```markdown
# 큰 제목 (H1)
## 중간 제목 (H2)
### 작은 제목 (H3)

**굵은 글씨**
*기울임 글씨*
~~취소선~~

- 글머리 목록
  - 하위 항목

1. 번호 목록
2. 두 번째

> 인용문 / 중요 메모

[링크 텍스트](https://example.com)
![이미지 설명](images/사진.png)

| 항목 | 내용 |
|------|------|
| 내용 | 내용 |

- [ ] 할 일 (체크박스)
- [x] 완료된 항목
```

> **팁:** 문법을 외울 필요 없습니다. 편집 모드 상단 **툴바 버튼**을 클릭하면 자동 삽입됩니다.

### 편집 모드 단축키

| 기능 | 단축키 |
|------|--------|
| 굵게 | `Ctrl+B` |
| 기울임 | `Ctrl+I` |
| 제목 1~3 | `Ctrl+1` / `2` / `3` |
| 링크 삽입 | `Ctrl+K` |
| 체크리스트 | `Ctrl+Shift+9` |
| 저장 | `Ctrl+S` |

### 편집 모드 종류

| 모드 | 설명 | 용도 |
|------|------|------|
| 편집 | 마크다운 코드만 보임 | 빠른 작성 |
| 분할 | 왼쪽 편집 + 오른쪽 미리보기 | 결과 확인하며 작성 |
| 미리보기 | 렌더링된 결과만 보임 | 검토·보고 |

## 5. 태그 활용법

파일 맨 위에 다음과 같이 작성하면 **태그 탭**에서 분류·검색할 수 있습니다.

```markdown
---
tags: [공문서, 수신, 2026, 진행중]
---

# 문서 제목
```

### 총무과 추천 태그 체계

| 분류 | 태그 예시 |
|------|----------|
| 문서 종류 | `공문서`, `회의록`, `보고서`, `계약서`, `점검일지` |
| 업무 분야 | `구매`, `시설`, `인사`, `예산`, `안전` |
| 상태 | `진행중`, `결재대기`, `완료`, `보류`, `반려` |
| 우선순위 | `긴급`, `일반` |

> **참고:** 상태 태그는 각 PC에만 저장됩니다. 팀 공유가 필요한 분류는 frontmatter를 사용하세요.

## 6. 칸반 보드로 업무 관리

좌측 **칸반 보드 탭**에서 업무를 카드로 시각화하여 관리합니다.

### 총무과 보드 구성 예시

```
보드: "총무과 업무현황"
├── 접수/요청
│   └── [카드] 소화기 교체 견적 요청 (담당: 팀원C, 마감: 4/10)
├── 처리중
│   └── [카드] 4월 비품 구매 (담당: 팀원A, 파일: 비품구매요청서.md)
├── 결재대기
│   └── [카드] 청소 용역 계약 갱신 (담당: 팀원A)
└── 완료
    └── [카드] 1분기 안전점검 보고 → 자동 아카이브
```

### 카드 구성 항목

| 항목 | 내용 |
|------|------|
| 제목 | 업무 내용 |
| 담당자 | 담당 팀원 이름 |
| 마감일 | 처리 기한 |
| 라벨 | 긴급 / 문서 / 시설 / 구매 등 |
| 연결 파일 | 관련 마크다운 파일 연결 |

> **팁:** 카드를 드래그하여 컬럼 간 이동, 완료된 업무는 자동 아카이브 설정으로 정리됩니다.

## 7. Git 사용법 (팀 동기화)

### Git이란?

파일의 변경 이력을 관리하고, 여러 PC에서 동기화할 수 있는 도구입니다.
Docuflow에서는 좌측 **Git 탭**의 버튼 클릭만으로 사용할 수 있습니다.

### 초기 설정 (최초 1회, 과장 또는 담당자)

#### 방법 A: 새로 시작하는 경우

1. 프로젝트 헤더 우클릭 → **"Git 저장소 초기화"**
2. Git 탭 → **원격 저장소 URL 입력**
3. 모든 파일 Stage → 커밋 메시지 입력 → 커밋 → Push

#### 방법 B: 이미 있는 저장소를 받아오는 경우

1. Git 탭 상단 **"Clone"** 버튼 클릭
2. GitHub/GitLab URL 입력
3. 저장할 상위 폴더 선택 → 자동으로 Clone + 프로젝트 등록

### 일상 업무 흐름

```
[출근 시]
  Git 탭 → Pull 버튼(↓) 클릭
  → 다른 팀원이 올린 최신 문서 다운로드

[업무 중]
  파일 작성·수정 → 파일 트리에서 변경 표시(M, U) 확인

[퇴근 전 또는 문서 완성 시]
  Git 탭 →
  1. 변경된 파일 확인
  2. "모두 스테이지" 클릭
  3. 커밋 메시지 입력 (예: "4월 비품구매 요청서 작성")
  4. "저장(커밋)" 클릭
  5. Push 버튼(↑) 클릭 → Git 탭에서 Push
```

> **빠른 Pull:** 프로젝트 탭 또는 칸반 탭의 프로젝트 이름 옆 **↓ 버튼**을 클릭하면 바로 받아올 수 있습니다.

### Git 상태 표시

| 표시 | 색상 | 의미 |
|------|------|------|
| **U** | 초록 | 새 파일 (아직 커밋 안됨) |
| **M** | 주황 | 수정됨 |
| **A** | 초록 | 스테이지됨 |
| **D** | 빨강 | 삭제됨 |

### 문제 해결

| 상황 | 해결 방법 |
|------|----------|
| Push가 안 될 때 | 먼저 Pull → 충돌 확인 후 다시 Push |
| 같은 파일 충돌 | 담당 파일을 명확히 나누거나, 한 명이 수정 후 공유 |
| 실수로 파일 삭제 | Git 탭 → 이력 → 되돌리기(↩) |

## 8. 검색 기능

### 파일명 검색

좌측 프로젝트 탭 상단 **"파일명 검색"** 입력란에 파일명 일부를 입력하면 트리에서 필터링됩니다.

### 내용 전문 검색

**"내용 전문 검색"** 입력란에 키워드를 입력하면 모든 프로젝트 파일 내용에서 검색합니다.

검색 대상:
- 마크다운 (.md) 파일
- 워드 (.docx, .doc) 파일

### 태그 검색

좌측 **태그 탭**에서 태그를 클릭하면 해당 태그가 있는 문서만 필터링됩니다.

### 업무별 활용 예시

```
프롬프트 예시:
"105병상 종합병원 총무과에서 발신하는 공문을 작성해줘.
수신: 소방서
제목: 소방 시설 점검 협조 요청
내용: 연 2회 정기 점검 일정 협의 요청
마크다운 형식으로 작성해줘."
```

#### 회의 내용 정리

```
프롬프트 예시:
"다음 메모를 총무과 회의록 형식으로 정리해줘:
- 날짜: 2026-04-01
- 참석: 과장, 팀원 3명
- 내용: (메모 붙여넣기)
결정사항과 담당자/기한을 표로 정리해줘."
```

#### 계약서 검토 요약


## 12. 자주 묻는 질문

### Q: Word 파일로 제출해야 하는 경우는?
A: Docuflow에서 마크다운으로 작성 후, 필요 시 Word에서 별도 편집하거나 복사·붙여넣기로 옮기세요. Docuflow에서 .docx 파일도 미리보기와 검색이 가능합니다.

### Q: 파일을 실수로 삭제했어요
A: Git을 사용 중이라면 Git 탭 → 이력 → 되돌리기(↩)를 클릭하세요. 휴지통에서도 복구할 수 있습니다.

### Q: 다른 PC에서 작업하려면?
A: Git 탭 → Clone → 저장소 URL 입력. 동일한 파일을 어느 PC에서든 받아올 수 있습니다.

### Q: 인터넷이 안 되면 사용 못 하나요?
A: 로컬 편집·저장은 정상 작동합니다. Push/Pull만 인터넷이 필요합니다.

### Q: 기안·결재 시스템(전자결재)과 함께 쓸 수 있나요?
A: Docuflow는 초안 작성 및 내부 공유 도구입니다. 최종 결재는 병원 전자결재 시스템을 이용하고, 결재된 문서의 사본을 Docuflow에 보관하는 방식을 권장합니다.

### Q: 칸반 보드 데이터는 공유되나요?
A: 프로젝트 폴더 내 `.kanban/` 폴더에 저장됩니다. Git으로 동기화하면 팀원과 보드를 공유할 수 있습니다.
