# 승인 워크플로우 샘플 문서

이 폴더의 문서들은 Docuflow 승인 워크플로우 기능을 시연하기 위한 샘플입니다.

## 사용 방법

1. Docuflow 좌측 사이드바에서 **프로젝트 폴더 추가** 클릭
2. 이 `samples/workflow` 폴더를 선택
3. 좌측 활동바의 **✓ (승인 워크플로우)** 아이콘 클릭
4. 상단에서 **현재 사용자** 이름을 설정 — 아래 이름 중 하나를 입력하면 실제 받은 요청이 보입니다

## 샘플 문서 구성

| 파일 | 상태 | 작성자 | 리뷰어 | 승인자 |
|---|---|---|---|---|
| 01_초안_... | 📝 초안 | 박재범 | (미지정) | (미지정) |
| 02_검토중_... | 👀 검토중 | 박재범 | 김리뷰 ✅ / 이검토 ⏳ | 최승인 ⏳ / 정법무 ⏳ |
| 03_승인됨_... | ✅ 승인됨 | 박재범 | 김리뷰 ✅ / 이재무 ✅ | 최대표 ✅ |
| 04_반려됨_... | ❌ 반려됨 | 박재범 | 김리뷰 ✅ / 이인사 ❌ | 최대표 ⏳ |

## 역할 시뮬레이션

사용자 이름을 바꿔가며 각 역할에서 시스템이 어떻게 동작하는지 확인하세요:

- **`박재범`** (작성자) — "내 초안", "내가 요청한 리뷰" 섹션 확인 가능
- **`이검토`** — 02번 문서가 "받은 요청"에 표시됨 (리뷰 대기)
- **`최승인`** — 02번 문서의 리뷰어가 모두 완료되지 않았으므로 아직 "받은 요청"에 표시되지 않음
- **`김리뷰`** — 이미 승인한 문서들에 대한 기록 확인 가능

## 확인 포인트

- 사이드바 ✓ 아이콘 우측의 **빨간 배지 숫자** = 받은 요청 수
- FileTree에서 파일명 옆 **상태 이모지** + 액션 필요 시 **빨간 점**
- **대시보드**: 4칸 칸반 뷰 + "받은 요청" 필터 + 빨간 테두리 카드
- 문서를 열고 우측 **"승인" 탭** → 리뷰/승인 액션
