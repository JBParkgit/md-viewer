# Docuflow 활용 가이드

## Hospital RUN 마케팅팀을 위한 문서 관리 & AI 협업 가이드

---

## 1. Docuflow란?

Docuflow는 마크다운(.md) 문서를 중심으로 파일을 관리하고, 편집하고, 팀과 공유할 수 있는 데스크톱 프로그램입니다.

### 왜 마크다운을 쓰나요?

| 기존 방식 | Docuflow 방식 |
|----------|-------------|
| Word로 작성 → 파일 전달 → 수정 → 재전달 | 마크다운으로 작성 → Git으로 동기화 |
| 서식에 시간 소모 | 내용에 집중, 서식은 자동 |
| AI 결과물을 다시 정리해야 함 | AI 결과물(마크다운)을 그대로 사용 |

### 주요 기능

- 마크다운 편집기 + 실시간 미리보기
- 프로젝트(폴더) 단위 파일 관리
- Word(.docx), PDF, 이미지, 동영상 미리보기
- 태그 기반 문서 분류 및 검색
- **칸반 보드** — 업무를 카드로 관리하고 파일과 연결
- Git 연동 (저장/올리기/받기)
- 전문 검색 (마크다운 + 워드 문서 내용)

### 사이드바 탭 구성

| 탭 | 기능 |
|----|------|
| 프로젝트 | 폴더 구조 탐색, 파일 생성/이동, 파일명·전문 검색 |
| 즐겨찾기 | 별표 추가한 파일 빠른 접근 |
| 최근 파일 | 최근에 열어본 파일 목록 |
| 태그 | 프론트매터 태그 기반 분류 및 색상 관리 |
| 갤러리 | 프로젝트 내 이미지·동영상 썸네일 모아보기 |
| 칸반 보드 | 프로젝트별 업무 보드 관리 |
| Git | 커밋·푸시·풀 등 버전 관리 |

## 2. 설치 및 실행

### 설치

1. `Docuflow` 폴더를 원하는 위치에 복사
2. `Docuflow.exe` 실행
3. 바탕화면 바로가기 생성 권장

### 첫 실행 시 설정

1. 상단 "프로젝트 추가" 버튼 클릭
2. 마케팅팀 공유 폴더 선택
3. 좌측 트리에서 파일 탐색 시작

## 3. 프로젝트 구성

### 추천 폴더 구조

```
마케팅_콘텐츠/
  ├── 블로그/
  │   ├── 관절/
  │   ├── 척추/
  │   └── 건강정보/
  ├── SNS/
  │   ├── 2026-03/
  │   ├── 2026-04/
  │   └── 콘텐츠_캘린더.md
  ├── 뉴스레터/
  ├── 영상스크립트/
  └── images/

마케팅_기획/
  ├── 회의록/
  ├── 캠페인/
  ├── 보고서/
  ├── 기획안/
  └── 예산/

마케팅_자료실/
  ├── 브랜드가이드/
  ├── 발표자료/
  ├── 템플릿/
  └── 레퍼런스/
```

### 담당자별 역할

| 담당자 | 주요 폴더 | 업무 |
|--------|----------|------|
| 팀장 | 기획/, 보고서/, 회의록/ | 전략 기획, 보고서, 회의록 |
| 콘텐츠 담당 | 블로그/, 뉴스레터/ | 블로그 글, 뉴스레터 작성 |
| SNS 담당 | SNS/, 영상스크립트/ | SNS 콘텐츠, 영상 기획 |
| 디자인 담당 | 자료실/, images/ | 디자인 자산, 발표자료 |

> **충돌 방지 팁:** 같은 파일을 동시에 수정하면 충돌이 발생할 수 있습니다. 담당 폴더를 나누어 작업하세요.

## 4. 마크다운 기본 사용법

### 자주 쓰는 문법

```markdown
# 큰 제목 (H1)
## 중간 제목 (H2)
### 작은 제목 (H3)

**굵은 글씨**
*기울임 글씨*
~~취소선~~

- 글머리 기호 목록
- 두 번째 항목
  - 하위 항목

1. 번호 목록
2. 두 번째
3. 세 번째

> 인용문

[링크 텍스트](https://example.com)
![이미지 설명](images/사진.png)

| 제목1 | 제목2 |
|-------|-------|
| 내용1 | 내용2 |
```

### 단축키 (편집 모드)

| 기능 | 단축키 |
|------|--------|
| 제목 1 (H1) | `Ctrl+1` |
| 제목 2 (H2) | `Ctrl+2` |
| 제목 3 (H3) | `Ctrl+3` |
| **굵게** | `Ctrl+B` |
| *기울임* | `Ctrl+I` |
| ~~취소선~~ | `Ctrl+Shift+S` |
| `인라인 코드` | `Ctrl+E` |
| 링크 | `Ctrl+K` |
| 글머리 목록 | `Ctrl+Shift+8` |
| 번호 목록 | `Ctrl+Shift+7` |
| 체크리스트 | `Ctrl+Shift+9` |
| 인용문 | `Ctrl+Shift+Q` |
| 형광펜 | `Ctrl+Shift+H` |
| 코드 블록 | `Ctrl+Shift+E` |
| 이미지 | `Ctrl+Shift+I` |
| 구분선 | `Ctrl+Shift+-` |
| 저장 | `Ctrl+S` |

> **팁:** 마크다운 문법을 외울 필요 없습니다! 편집 모드의 **툴바 버튼**을 클릭하면 자동으로 삽입됩니다.

### 편집 모드 종류

| 모드 | 설명 | 용도 |
|------|------|------|
| 편집 | 마크다운 코드만 보임 | 빠른 작성 |
| 분할 | 왼쪽 편집 + 오른쪽 미리보기 | 결과 확인하며 작성 |
| 미리보기 | 렌더링된 결과만 보임 | 읽기/검토 |

## 5. 태그 활용법

### Frontmatter 태그 (팀 공유용)

파일 맨 위에 다음과 같이 작성하면 태그 탭에서 분류/검색할 수 있습니다:

```markdown
---
tags: [관절, 블로그, 진행중]
---

# 문서 제목

본문 내용...
```

### 추천 태그 체계

| 분류 | 태그 예시 |
|------|----------|
| 진료과 | `관절`, `척추`, `재활`, `건강검진`, `도수치료` |
| 채널 | `블로그`, `SNS`, `뉴스레터`, `보도자료`, `영상` |
| 상태 | `진행중`, `검토중`, `완료`, `보류` |
| 우선순위 | `긴급`, `일반` |

### 상태 태그 (개인용)

파일 우클릭 → 태그 관리에서 상태 태그를 지정할 수 있습니다:
- 진행중 / 승인완료 / 검토필요 / 초안 / 최종본

> **참고:** 상태 태그는 각 PC에만 저장됩니다. 팀 공유가 필요한 태그는 frontmatter를 사용하세요.

## 6. AI (Claude) 활용법

### Claude와 Docuflow 연동 워크플로우

```
1. Claude에게 작업 요청
   "관절 내시경 수술에 대한 블로그 글을 작성해줘"

2. Claude의 마크다운 응답을 복사
   (코드블록의 복사 버튼 사용 → 마크다운 형식 유지)

3. Docuflow에서 새 파일 생성
   프로젝트 + 버튼 → 파일명 입력

4. 편집 모드에서 붙여넣기
   Ctrl+V → 마크다운이 그대로 적용

5. 미리보기로 확인 → 수정 → 저장
```

### Claude 활용 시나리오

#### 블로그 글 작성

```
프롬프트 예시:
"105병상 관절 척추 전문병원의 블로그에 올릴 글을 작성해줘.
주제: 무릎 인공관절 수술 후 재활 방법
대상: 60대 이상 환자 및 보호자
톤: 전문적이면서 쉬운 설명
마크다운 형식으로 작성해줘."
```

#### 회의록 정리

```
프롬프트 예시:
"다음 회의 내용을 마크다운 회의록으로 정리해줘:
- 날짜: 2026-03-22
- 참석자: 마케팅팀 4명
- 내용: (회의 메모를 붙여넣기)
결정사항, 할일, 담당자를 표로 정리해줘."
```

#### SNS 콘텐츠 변환

```
프롬프트 예시:
"다음 블로그 글을 인스타그램 / 네이버 블로그 / 카카오톡 채널
각 플랫폼에 맞는 버전으로 변환해줘:
(블로그 글 내용 붙여넣기)"
```

#### 경쟁 분석

```
프롬프트 예시:
"우리 병원(관절 척추 전문, 105병상)과 경쟁하는
주변 정형외과/척추 병원들의 마케팅 전략을 분석해줘.
비교 표 형식으로 작성해줘."
```

#### SEO 키워드 리서치

```
프롬프트 예시:
"관절 척추 병원 블로그에 사용할 SEO 키워드를 추천해줘.
진료과목별로 분류하고, 검색량 예상도 포함해줘.
마크다운 표 형식으로 작성해줘."
```

### Claude Code (코드 모드) 활용

Claude Code를 사용하면 더 강력한 작업이 가능합니다:

- **대량 파일 처리**: 여러 마크다운 파일을 한번에 수정/변환
- **데이터 분석**: 마케팅 성과 데이터를 분석하여 보고서 자동 생성
- **템플릿 생성**: 반복 사용하는 문서 템플릿을 자동으로 생성
- **파일 정리**: 폴더 구조 정리, 파일명 일괄 변경

### 복사 시 주의사항

| 복사 방법 | 결과 |
|----------|------|
| Claude 코드블록 **복사 버튼** 클릭 | 마크다운 형식 유지 (추천) |
| 화면을 **드래그하여 복사** | 서식 없는 텍스트만 복사됨 |

## 7. Git 사용법 (팀 동기화)

### Git이란?

파일의 변경 이력을 관리하고, 여러 PC에서 동기화할 수 있는 도구입니다.
Docuflow에서는 버튼 클릭만으로 사용할 수 있습니다.

### 초기 설정 (최초 1회)

#### 방법 A: 새 프로젝트를 Git으로 시작

1. 프로젝트 헤더 우클릭 → **"Git 저장소 초기화"**
2. 좌측 Git 탭 → **"+ 원격 저장소 설정"**
3. GitHub 저장소 URL 입력 (예: `https://github.com/팀계정/마케팅콘텐츠.git`)
4. 모든 파일 Stage → Commit → Push

#### 방법 B: 기존 GitHub 저장소를 Clone

1. 좌측 Git 탭 → **"저장소 Clone"** 버튼
2. GitHub URL 입력
3. 저장할 폴더 위치 선택
4. 자동으로 Clone + 프로젝트 등록

### 일상 작업 흐름

```
[출근]
  Git 탭 → "받기 (Pull)" 클릭
  → 다른 팀원이 올린 최신 내용 다운로드

[업무 중]
  파일 수정 → 트리에서 변경 표시 확인 (M, U 등)

[퇴근 전]
  Git 탭 →
  1. "모두 스테이지" 클릭
  2. 커밋 메시지 입력 (예: "블로그 글 3편 추가")
  3. "저장 (커밋)" 클릭
  4. "올리기 (Push)" 클릭
```

### Git 상태 표시

파일 트리에서 변경 상태가 표시됩니다:

| 표시 | 색상 | 의미 |
|------|------|------|
| **U** | 초록 | 새 파일 (아직 커밋 안됨) |
| **A** | 초록 | 추가됨 (스테이지됨) |
| **M** | 주황 | 수정됨 |
| **D** | 빨강 | 삭제됨 |
| **R** | 파랑 | 이름변경 |

### Git 탭 기능

| 기능 | 설명 |
|------|------|
| 받기 (Pull) | 원격 저장소에서 최신 내용 다운로드 |
| 올리기 (Push) | 내 변경사항을 원격에 업로드 |
| 저장 (커밋) | 변경사항을 하나의 묶음으로 기록 |
| 스테이지 | 커밋할 파일 선택 |
| 되돌리기 | 이전 커밋 상태로 복구 (이력 유지) |

### 문제 해결

| 상황 | 해결 방법 |
|------|----------|
| Push가 안 될 때 | 먼저 Pull 후 다시 Push |
| 같은 파일 충돌 | 담당자를 정하고 한 명만 수정 |
| 실수로 파일 삭제 | Git 탭 → 최근 이력 → 되돌리기(↩) |

## 8. 칸반 보드

프로젝트별 업무를 카드로 관리하는 기능입니다. 좌측 **칸반 보드 탭**에서 사용합니다.

### 보드 구성

- **다중 보드**: 프로젝트당 여러 보드 생성 가능 (예: "3월 콘텐츠", "캠페인 기획")
- **컬럼**: 보드 안에 단계별 컬럼 추가 (예: 할 일 → 진행중 → 완료)
- **카드**: 각 컬럼에 업무 카드 생성

### 카드 구성 요소

| 항목 | 내용 |
|------|------|
| 제목 | 업무 내용 |
| 설명 | 상세 메모 |
| 담당자 | 담당 팀원 이름 |
| 마감일 | 날짜 지정 |
| 라벨 | 긴급 / 버그 / 기능 / 개선 / 문서 (색상 코드) |
| 연결 파일 | 프로젝트 내 파일 다중 연결 (클릭하면 파일 열림) |

### 카드 이동

드래그앤드롭으로 카드를 다른 컬럼으로 이동합니다. 컬럼 순서도 드래그로 변경할 수 있습니다.

### 아카이브

완료된 카드를 자동으로 보관합니다:

1. 보드 설정(⚙️)에서 아카이브 대상 컬럼 지정
2. 자동 아카이브 기간 설정 (예: 7일 후 자동 보관)
3. 보관된 카드는 아카이브 목록에서 복원하거나 삭제 가능

### 활용 예시

```
보드: "3월 블로그 콘텐츠"
├── 기획 중
│   └── [카드] 무릎 인공관절 수술 후 재활 (담당: 김OO, 마감: 3/25)
├── 작성 중
│   └── [카드] 척추관 협착증 증상 안내 (연결: 척추협착증_초안.md)
└── 완료
    └── [카드] 도수치료 효과 안내 → 자동 아카이브
```

## 9. 갤러리

좌측 **갤러리 탭**에서 모든 프로젝트의 이미지와 동영상을 한눈에 확인합니다.

- 지원 형식: PNG, JPG, GIF, WebP, SVG, MP4, MOV 등
- 프로젝트별로 접기/펼치기 가능
- 동영상은 재생 아이콘으로 구분
- 클릭: 미리보기 탭으로 열기 / 더블클릭: 고정 탭으로 열기
- 현재 열려있는 파일은 자동으로 하이라이트

## 10. 검색 기능

### 파일명 검색

좌측 프로젝트 탭의 **"파일명 검색"** 입력란에 파일명 일부를 입력하면 트리에서 필터링됩니다.

### 내용 전문 검색

**"내용 전문 검색"** 입력란에 키워드를 입력하면 모든 프로젝트의 파일 내용에서 검색합니다.

검색 대상:
- 마크다운 (.md) 파일
- 워드 (.docx, .doc) 파일

### 태그 검색

좌측 **태그 탭**에서 태그를 클릭하면 해당 태그가 있는 문서만 필터링됩니다.

## 11. 파일 관리 팁

### 단축키

| 키 | 기능 |
|---|------|
| 클릭 | 미리보기 탭으로 열기 |
| 더블클릭 | 고정 탭으로 열기 |
| Ctrl+클릭 | 복수 선택 |
| Shift+클릭 | 범위 선택 |
| Delete | 선택한 파일/폴더 삭제 |
| F2 | 이름 변경 |
| 드래그 & 드롭 | 파일/폴더 이동 (복수 선택 지원) |

### 이미지 관리

- 각 폴더 하위에 `images/` 폴더를 만들어 이미지 정리
- 마크다운에서 이미지 삽입: `![설명](images/파일명.png)`
- 갤러리 탭에서 프로젝트의 모든 이미지를 한눈에 확인

### 글꼴 & 글자 크기

- 상단 툴바에서 **글꼴 드롭다운**으로 글꼴 변경
- **A- / A+** 버튼으로 글자 크기 조절
- 설정은 PC에 저장되어 다음 실행 시에도 유지

## 13. 동기화 방식 비교

| 항목 | Git (GitHub) | OneDrive |
|------|-------------|----------|

| 버전 관리 | 전체 이력 영구 보존 | 30일 제한 |
| 대용량 파일 | 약함 (100MB 제한) | 강함 |

## 14. 자주 묻는 질문

### Q: 마크다운을 꼭 배워야 하나요?
A: 아닙니다. 편집 모드의 툴바 버튼을 클릭하면 됩니다. 또는 Claude에게 "마크다운으로 작성해줘"라고 요청하세요.

### Q: Word 파일은 어떻게 하나요?
A: Docuflow에서 미리보기와 내용 검색이 가능합니다. 편집은 Word에서 하세요 (더블클릭 또는 우클릭 → 기본 앱으로 열기).

### Q: 실수로 파일을 삭제했어요
A: Git을 사용 중이라면 Git 탭 → 최근 이력에서 되돌리기(↩)를 클릭하세요. Delete 키로 삭제한 경우 휴지통에서 복구할 수 있습니다.

### Q: 다른 PC에서 작업하려면?
A: GitHub에 Push한 후, 다른 PC에서 Docuflow 설치 → Git 탭 → Clone → 같은 GitHub URL 입력.

### Q: 인터넷이 안 되면?
A: 로컬에서 편집/저장은 정상 작동합니다. Push/Pull만 인터넷이 필요합니다. 연결되면 Push하세요.

### Q: 칸반 보드 데이터는 어디에 저장되나요?
A: 프로젝트 폴더 내 `.kanban/` 폴더에 JSON 파일로 저장됩니다. Git으로 동기화하면 팀원과 보드를 공유할 수 있습니다.

### Q: 프로그램이 종료되지 않아요
A: 저장하지 않은 문서가 있으면 확인 대화상자가 나타납니다. 저장 후 종료하세요.
